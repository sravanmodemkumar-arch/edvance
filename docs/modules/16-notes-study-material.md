# Module 16 — Notes & Study Material

## Purpose
Provide a three-tier, platform-native notes and study material system for EduForge.

**Tier 1 — Platform Content Library:** Pre-loaded by the EduForge content team covering
all major boards (CBSE, ICSE, all state boards), competitive exams (JEE, NEET, UPSC, CA,
Banking, Law), and vocational curricula (ITI/NCVT). Students and teachers get ready-to-use
content from day one — no waiting for teachers to create notes.

**Tier 2 — Institution Notes Library:** HOD-approved notes created and maintained by the
institution's own teachers; shared across all teachers of the same subject/board/grade.

**Tier 3 — Teacher / Batch Notes:** Teacher-created notes scoped to their own section or
coaching batch; most granular and personalised.

No file uploads. No downloads. YouTube links open in-app WebView via "Watch Video" button.
All content is created, stored, and consumed within the platform.

---

## 1. Content Hierarchy — Six Levels

Every note in the system — whether platform-provided, institution-authored, or
teacher-created — is linked to a precise location in the six-level content hierarchy:

```
Level 1 → Board / Framework
Level 2 → Subject
Level 3 → Grade / Semester / Year
Level 4 → Unit
Level 5 → Chapter
Level 6 → Topic  (and optionally Sub-topic — Level 6b)
```

### 1.1 Navigation Structure (Student & Teacher View)

```
Board: CBSE
  └── Subject: Physics
        └── Grade: Class 12
              └── Unit 1: Electrostatics
                    └── Chapter 1: Electric Charges and Fields
                          ├── Topic: Electric Charge — Properties
                          │     ├── Sub-topic: Conservation of Charge
                          │     ├── Sub-topic: Quantisation of Charge
                          │     └── Sub-topic: Additivity of Charge
                          ├── Topic: Coulomb's Law
                          │     ├── Sub-topic: Statement and Formula
                          │     ├── Sub-topic: Vector Form
                          │     └── Sub-topic: Comparison with Gravitational Force
                          └── Topic: Electric Field
                                ├── Sub-topic: Definition and SI Unit
                                ├── Sub-topic: Electric Field Lines
                                └── Sub-topic: Electric Field due to Point Charge
```

### 1.2 Notes at Each Level

| Level | Allowed Note Types |
|---|---|
| Unit | Unit Overview, Unit Formula Sheet, Unit Mind Map |
| Chapter | Chapter Summary, Chapter Formula Sheet, Chapter PYQ Analysis, Chapter Revision Pack, Worked Examples |
| Topic | Lecture Notes, Concept Explanation, Common Mistakes, Exam Tips, Worked Examples |
| Sub-topic | Detailed Explanation, Step-by-step Derivation, Numerical Solved, Short Notes |

Notes at a higher level (unit/chapter) are visible when student browses that level.
Notes at topic/sub-topic level are the most granular — most content lives here.

### 1.3 Drill-Down View (Student)
- Student opens Subject → sees Unit list with note count per unit
- Opens Unit → sees Chapter list with note count per chapter
- Opens Chapter → sees Topic list; chapter-level notes shown at top
- Opens Topic → sees Sub-topic list; topic-level notes shown at top
- Opens Sub-topic → sees all notes for that sub-topic
- Each level shows: Platform notes count | Institution notes count | Teacher notes count
- Student can filter by note type or by source (Platform / Institution / Teacher)

## 5. Rich Text Editor — Full Specification

### 5.1 Core Formatting
- Bold, Italic, Underline, Strikethrough
- Heading H1 / H2 / H3 / H4
- Bullet list (unordered), numbered list (ordered), nested lists (3 levels deep)
- Blockquote, horizontal rule, inline code, code block
- Superscript / Subscript (chemical formulas, footnotes, mathematical notation)
- Text colour (limited 8-colour palette), highlight colour (yellow / green / pink)
- Alignment: Left / Centre / Right / Justify

### 5.2 Math Equation Editor
- LaTeX inline: `$E=mc^2$` and block: `$$\oint \vec{E} \cdot d\vec{A} = \frac{Q_{enc}}{\varepsilon_0}$$`
- Live rendered preview while typing
- Equation palette: Greek letters (α β γ δ…), operators (∑ ∫ ∂ ∇…), set notation (∈ ⊂ ∩ ∪…), calculus, statistics, matrices
- Fraction, root, power, subscript quick-insert
- Same engine as Module 14 (unified across platform)

### 5.3 Chemical Notation
- Unicode subscript/superscript for formulas: H₂SO₄, CH₃COOH, CaCO₃, Na₂CO₃
- IUPAC name field per compound
- Structural formula: condensed structural text notation
- Chemical equation: arrow notation → ⇌ with conditions above/below arrow
- Ionic equation: charge notation (Cu²⁺, SO₄²⁻)

### 5.4 Diagram / Sketch Canvas
- In-platform only — no image upload from device storage
- Tools: line, arrow (single/double), rectangle, circle, ellipse, polygon, freehand brush
- Text label with leader line (callout)
- Colour fill and border colour per shape
- Use cases: biology diagrams, physics circuit diagrams, geometry constructions,
  geographic map sketches, flowcharts, block diagrams, Venn diagrams
- Canvas auto-scales for mobile/tablet/web

### 5.5 Table Builder
- Define rows × columns (up to 20×20)
- Merge cells horizontal and vertical
- Header row highlight, header column highlight
- Column width adjustment
- Alignment per cell
- Use cases: comparison tables, periodic data, truth tables, case study comparison,
  formula reference tables, marks distribution tables

### 5.6 Code Block
- Syntax highlighting: Python, C, C++, Java, JavaScript, HTML, CSS, SQL, Scratch pseudocode
- Line numbering, monospace font
- Copy-to-clipboard for teacher preview (students see view-only)

### 5.7 Watch Video Button (YouTube & Platform-Hosted)
- Teacher or platform content editor pastes video URL (YouTube or platform CDN)
- System renders inline "Watch Video" button with custom label
- **YouTube:** opens in in-app WebView — YouTube app never launched, user stays in EduForge

## 6. Note Types — Detailed Specifications

### 6.1 Lecture Notes
- Full-period lecture in structured sections
- Objectives at top (from Module 15 topic LOs)
- Recap section: links to prerequisite topic notes
- Main content: paragraphs + inline diagrams + equations
- Worked example embedded inline
- Summary box at bottom: 5 key takeaways
- Teacher's note field (hidden from students — for teacher's own reference)

### 6.2 Summary Notes
- Pre-filled section headings from Module 15 topic list for the chapter
- Each section: 3–5 bullet points maximum
- Bold key terms
- No lengthy paragraphs — only crisp statements
- Auto word-count indicator: advisory max 500 words per chapter

### 6.3 Revision Notes (Pre-Exam)
- 1-page per chapter target (enforced as advisory word limit)
- Only high-frequency exam points
- Bloom's L1–L2 focus (Remember / Understand)
- Includes: 3 most-asked exam questions with one-line answers
- Released 2 weeks before exam (scheduled release)

### 6.4 Formula Sheet
- Chapter-wise two-column table: Formula Name | Formula (LaTeX-rendered)
- SI unit column, Conditions/applicable range column
- Sorted by topic sequence
- Subject-wide consolidated formula sheet (all chapters in one note)
- Print-friendly: A4 single page with reduced font for comprehensive coverage

### 6.5 Mind Map
- Root node = chapter/topic name
- Main branches = sub-topics
- Leaf nodes = key terms, definitions, formulae, examples
- Colour per branch (auto-assigned; editable)
- Expandable/collapsible in student view
- Each node: max 10-word label

### 6.6 Concept Map
- Free-form canvas
- Nodes = concepts
- Directed arrows with relationship labels
- Shows how concepts interconnect (not hierarchical like mind map)
- Used for cross-topic and cross-chapter concept links

### 6.7 Worked Examples
Structured format:
```
Problem Statement: [Full problem text with equations]
Given:            [All known quantities with units]
Find:             [What to determine]
Approach:         [Method / formula to use]
Solution:
  Step 1: [Description] — [Equation / calculation]
  Step 2: …
  Step N: …
Answer:           [Final answer with units, sig figs]
Variation:        [Same problem with different numbers or approach]
Note:             [Common error students make here]
```

### 6.8 Common Mistakes
- Two-column table per mistake
- Wrong Approach column: shows incorrect student answer (as commonly written in exams)
- Correct Approach column: shows correct method/answer
- Source: based on CBSE/board examiner reports and marking scheme analysis
- Warning icon next to frequent mistakes

### 6.9 Exam Tips
- Numbered tips per chapter
- Content: CBSE board answer format tips, word count guidance, diagram labelling tips,
  units to always mention, marks allocation strategy (1-mark vs 3-mark vs 5-mark answers)
- Source: CBSE examiner reports, marking scheme analysis
- Warning: "Frequently penalised" tag on tips for most-penalised mistakes

### 6.10 PYQ Analysis Note
- Chapter-wise frequency table: columns = last 10 years, rows = topics
- Cell: question count / marks asked in that year from that topic
- Heatmap colouring: dark = high frequency; light = low frequency
- Bottom row: Total marks, Importance rank (High / Medium / Low)
- Student strategy: "Focus Areas: Topics X, Y, Z appear in 7 of last 10 years"
- Source: platform content team entry from official board question papers

### 6.11 NCERT Solutions
- Intext questions: numbered per page (e.g., "Page 15, Q1")
- Exercise questions: numbered per chapter exercise
- Additional exercise / supplementary questions
- Exemplar questions (separate note)
- Format: Question → Model Answer (worked example for numerical; point-wise for theory)
- Aligned to CBSE marking scheme word limits

### 6.12 Derivation Notes
- All CBSE/board-prescribed derivations listed per chapter
- Each derivation: Statement → Diagram (canvas) → Step-by-step mathematical derivation → Result
- "Board Important" tag on derivations that appear in 5+ years of board exams
- Common mistakes in derivations noted at end

### 6.13 FLN Activity Card (Classes 1–3)
- Large font (18pt minimum)
- Concept explanation in simple language (max 12 words per sentence)
- Picture description: text description of a visual concept (no image upload)
- Activity steps: numbered, action words ("Draw", "Count", "Say aloud", "Circle")
- "DO THIS:" section: physical/oral activity the student performs
- NIPUN Bharat competency code linked
- Available in: English + Hindi + State regional language (trilingual)

### 6.14 Parent-Facing Weekly Summary (Classes 1–5)
- Auto-generated from teacher's lecture notes of the week
- Simplified language: "Your child is learning…" framing
- 3–5 key concepts in plain language (no jargon)
- "How to help at home:" — 2 practical activities
- Visible in parent app (Module 09) for parents of Classes 1–5 only
- School admin configures: on/off per grade

### 6.15 Guest Lecture Notes
- Linked to guest faculty account (Module 08)
- Notes scoped to single guest lecture session
- Auto-archived 7 days after session date
- Teacher can promote to institution library before archive if content is valuable

## 8. Student Experience

### 8.1 Homepage — Subject Notes Summary
- Each subject shows: total notes available (Platform + Institution + Teacher)
- Unread count per subject (badge)
- Last viewed note with "Continue Reading" shortcut
- "Exam Prep" section: revision packs for upcoming exams

### 8.2 Note Reading — In-App View
- Full-screen reading mode
- Scroll, pinch-to-zoom
- Full-text search within note (highlights all matches, navigate with arrows)
- Table of Contents: auto-generated for notes with > 5 sections (anchor links)
- Font size: 3 levels (Small / Medium / Large) — saved per student
- Night mode: renders per device dark-mode setting automatically
- **No download option anywhere — view-only in-app at all times**

### 8.3 Offline Cache
- Notes opened at least once are automatically cached — no manual action by student
- Cached notes readable fully offline with all formatting (equations, tables, diagrams)
- Cache updates automatically when student reconnects and a newer version exists
- System manages cache size: oldest un-bookmarked notes evicted first
- **Not a download — transparent background caching by the app**

### 8.4 Personal Tools (Student)
- **Bookmark:** tap bookmark icon → saved to personal Bookmarks list (private)
- **Highlight:** long-press → select text → choose colour (yellow / green / pink) → saved per student
- **Personal annotation:** tap paragraph → add typed note in margin → private
- All personal tools are private — not visible to teacher or classmates
- Personal tools retained across sessions; cleared only by student explicitly

### 8.5 Mark as Read
- "Mark as Read" button at bottom of note
- System also auto-marks if student scrolls to end + minimum time threshold met
- Read status: shown in chapter index, feeds teacher read-tracker dashboard

  - "Read before next chapter" badge on the note
  - When prerequisite chapter unlocks for student, reminder shown: "Review [Note Title] first"

### 8.7 Gamification

- Notes Completion badge: when student reads all notes for a chapter
- Visible in student profile; does not affect academic marks

### 8.8 Student Feedback on Notes
- Thumbs up / Thumbs down + optional one-line comment
- Student can choose anonymous (DPDPA 2023 compliant)
- Aggregated feedback visible to teacher/platform team (anonymous when student chose it)

### 8.9 CWSN Student Experience
- CWSN students (Module 07 profile flag) automatically shown CWSN variant note if available
- Screen reader compatible (ARIA semantic markup): TalkBack (Android) / VoiceOver (iOS)
- Font size always at Large by default for CWSN accounts
- Platform CWSN notes: simplified language, larger diagrams, shorter sections

## 11. Versioning & Quality Control

### 11.1 Auto-Versioning
- Every save = new version (v1, v2, v3…)
- Diff view: what changed between any two versions
- Restore any version (creates new version at top)
- Platform notes: versioned independently; institution content unaffected when platform updates

### 11.2 HOD Approval Workflow
- Teacher submits note → HOD queue → Approve / Return with inline comments
- Approved → available in institution library
- Return → teacher sees comments highlighted in note; re-submits after edits
- Bypass allowed for teacher's own section notes (no approval needed for section-scope)

### 11.3 Platform Content Quality Review
- Platform notes reviewed annually by SME + editorial team
- Student feedback aggregated: notes with < 3-star average flagged for review
- Teacher feedback: teachers can flag platform notes for inaccuracy → reviewed by platform team within 5 working days

### 11.4 Duplication Detection
- Before publishing: similarity check against institution library (> 80% → warning)
- Not applied to platform notes (platform is authoritative source)

### 11.5 Stale Notes Alert
- Notes > 1 academic year old flagged for HOD review at start of new year
- Teacher action: Archive / Update / Carry Forward
- Carry Forward clones note into new academic year with fresh version history

## 15. Data Architecture

### 15.1 Tenancy & Platform Separation
- Platform notes: `tenant_id = NULL` (global); all institutions can read, none can edit
- Institution notes: `tenant_id = institution_id` (RLS scoped)
- Teacher notes: `tenant_id = institution_id` + `scope = SECTION/BATCH` (RLS scoped)
- Student personal data (highlights, annotations, bookmarks): tenant_id scoped + student_id scoped

### 15.2 Database Schema

```sql
-- Notes master
notes (
  note_id          UUID PK,
  tenant_id        UUID FK tenants NULL,        -- NULL = platform note
  branch_id        UUID FK branches NULL,

  board_id         UUID FK boards NULL,          -- for platform notes
  subject_id       UUID FK subjects_master NULL,
```
