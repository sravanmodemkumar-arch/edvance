## Auto-Provisioning — Under 5 Minutes

After EduForge creates the tenant:

```
Step 1 — Assign shard
  Least-full available shard assigned automatically

Step 2 — Create DB schema
  Tenant schema created on assigned shard

Step 3 — CDN namespace
  Cloudflare R2 bucket created for this tenant

Step 4 — Subdomain
  xyz.eduforge.in provisioned automatically

Step 5 — Default config
  Placeholder logo, default colors, default theme applied

Step 6 — Module flags
  Modules enabled exactly per signed agreement
```

## Institution-Specific Configurations

### School
- Class + section grouping
- House system — sports, cultural competition tracking
- Subject teacher per class
- Class teacher per section
- Attendance: per day
- Parent access: full

### Intermediate College
- Stream selection: Science (PCM / PCB), Commerce, Arts
- Subject combination per stream — electives allowed
- Attendance: per day or per subject — college decides
- Board exam: internal marks by institution, external by state board
- Parent access: full — students are minors

### Degree / Engineering / Medical / Law College
- Semester system — separate results per semester
- Multiple courses — own syllabus per course
- HOD per department
- Elective subjects — timetable adjusts automatically
- Internal marks by college, external by university
- Attendance: per day or per subject — college decides
- Exam: per subject — one subject per exam day
- Backlog: failed subject tracked — next semester exam
