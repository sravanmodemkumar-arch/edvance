# EduForge — Module 07: Student Enrolment & Profile

> Every student on EduForge has a complete, verified profile from day one.
> Enrolment follows Indian national and state standards — Aadhaar, APAAR, RTE, reservations,
> CBSE/State Board age norms — all enforced at the platform level.

---

## Enrolment Types

| Type | Description | When Used |
|---|---|---|
| New Admission | Fresh student joining for first time | Start of academic year / mid-year |
| Lateral Entry | Joining mid-course — skipping initial year | Engineering 2nd year direct entry |
| Transfer from Other Institution | Student coming with TC from another school/college | Any time |
| Re-admission | Student who left and is rejoining | After gap year, withdrawal reversal |
| Bulk Enrolment | Entire batch created at academic year start | New academic year |

---

## Enrolment Process Flow

```
Step 1 — Admission enquiry received (Module 31 — CRM)
       │
Step 2 — Documents collected + verified (originals checked)
       │
Step 3 — Admin creates student profile
       │
Step 4 — Admission number auto-generated
       │
Step 5 — Class, section, roll number assigned
       │
Step 6 — Fee account created — installment schedule linked
       │
Step 7 — ID card generated
       │
Step 8 — Parent account linked
       │
Step 9 — Login credentials created — sent to student + parent via email
       │
Step 10 — Student is active — appears in attendance, timetable, all modules
```

## Unique Identifiers

| Identifier | Description | Format | Generated By |
|---|---|---|---|
| Admission Number | Unique per student per institution | Institution code + Year + Sequence | System auto-generates |
| Roll Number | Per class per section per academic year | Section code + Sequence | System auto-generates |
| Student ID Number | Indian standard — on ID card | College Code + Course Code + Year + Sequence | System auto-generates (Module 04) |
| Aadhaar Number | National ID | 12-digit | Admin enters — student provides |
| APAAR ID | Academic Bank of Credits — NEP 2020 | 12-digit | Admin enters if available |

- Admission number: permanent — never changes even if student transfers class
- Roll number: new each academic year
- Aadhaar: not mandatory — stored encrypted — DPDPA compliant

## Age Verification — Class-wise Indian Standards

| Class | Minimum Age | Maximum Age | Standard |
|---|---|---|---|
| Nursery / LKG | 3 years | 4 years | CBSE / Delhi |
| Class 1 | 5 years 6 months | 7 years | CBSE norm |
| Class 6 | 10 years | 12 years | General |
| Class 9 | 13 years | 15 years | General |
| Class 11 | 15 years | 17 years | General |
| Degree 1st year | 17 years | No upper limit | UGC |

## Student Status Lifecycle

| Status | Trigger | Access |
|---|---|---|
| Enrolled | Admission complete | Full access |
| Active | Academic year started | Full access |
| Inactive | Unexplained absence > 30 consecutive days | Read only — admin notified |
| Detained | Failed — retained in same class | Active in same class next year |
| Transferred | Inter-branch transfer complete | New branch access |
| Withdrawn | Student withdrew voluntarily | No access — data preserved |
| TC Issued | Transfer certificate issued | No access — data archived |
| Re-admitted | Withdrawn student rejoins | Full access restored |

### Student Inactivity Handling

| Trigger | Action |
|---|---|
| 7 days absent — no information | Attendance flagged — class teacher notified |
| 15 days absent — no information | HOD notified — parent contacted |
| 30 days absent — no information | Principal review — status → Inactive |
| 60 days absent | Admin initiates withdrawal process |
