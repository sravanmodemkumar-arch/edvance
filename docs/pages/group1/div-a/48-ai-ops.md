## 2. Purpose & Business Logic

EduForge uses AI/LLM for:
1. **MCQ Auto-generation** — institutions request AI-generated questions for their question bank. OpenAI GPT-4o or Anthropic Claude 3.5 Sonnet generates questions from topic prompts; a quality gate (human expert review) accepts/rejects each batch.
2. **Answer Explanation Generation** — post-result explanations for each answer option (why correct, why others wrong).
3. **Performance Analysis** — per-student weak-area reports generated by analysing exam attempt patterns.
4. **Content Tagging** — auto-tagging uploaded PDFs/videos by topic, difficulty, domain.

**Key operational risks:**
1. **MCQ quality rate** — if AI generates low-quality questions (rejected by reviewer), institution is blocked and files a support ticket
2. **Cost per question** — LLM API costs are input × output tokens. At scale (100K MCQs/month), cost control matters
3. **Pipeline queue depth** — generation jobs queued in Celery. If queue depth spikes (> 1,000 jobs), institutions experience delays > 30 min
4. **Model version drift** — if OpenAI deprecates a model version silently, generation fails
5. **API error rate** — OpenAI/Anthropic rate limits, 5xx errors, timeout failures

## 8. Backend View & API

```python
class AIOperationsView(LoginRequiredMixin, PermissionRequiredMixin, View):
    permission_required = "portal.view_ai_operations"

    def get(self, request):
        if request.user.role not in {"cto","ai_engineer","superadmin"}:
            return redirect("exec:dashboard")

        r = get_redis_connection()
        cache_key = "ai:ops:summary:v1"
        ctx = json.loads(r.get(cache_key) or "{}")
        if not ctx:
            ctx = self._build_ctx()
            r.setex(cache_key, 115, json.dumps(ctx))

        if _is_htmx(request):
            part = request.GET.get("part","")
            dispatch = {
                "summary":  "exec/ai/partials/summary.html",
                "pipeline": "exec/ai/partials/pipeline.html",
                "quality":  "exec/ai/partials/quality.html",
                "cost":     "exec/ai/partials/cost.html",
                "models":   "exec/ai/partials/models.html",
                "workers":  "exec/ai/partials/workers.html",
            }
            if part in dispatch:
                return render(request, dispatch[part], ctx)
        return render(request, "exec/ai_ops.html", ctx)
```

**Action endpoints:**

| Method | URL | Permission | Action |
|---|---|---|---|
| POST | `/exec/ai-ops/actions/pause-pipeline/` | CTO/AI Eng | Set `ai:pipeline:paused` Redis flag → workers stop picking new jobs |
| POST | `/exec/ai-ops/actions/resume-pipeline/` | CTO/AI Eng | Clear `ai:pipeline:paused` flag |
| POST | `/exec/ai-ops/actions/switch-model/` | CTO/AI Eng | Update `AIModelConfig.active_model` for task |

## 10. Security Considerations

- Topic prompts may contain institution-specific curriculum content — logs stored encrypted at rest
- API keys (OpenAI, Anthropic): in AWS Secrets Manager, rotated quarterly
- Pipeline pause is a high-impact action (blocks all institutions from AI features) — logged to `AuditLog` with reason field
- Cost overage auto-pause: prevents unbounded spend from runaway jobs
- Generated MCQs containing factually incorrect statements about regulated topics (medical, legal): reviewer workflow is mandatory for NEET/CA domains

## 11. Edge Cases

| State | Behaviour |
|---|---|
| OpenAI API rate limit hit | Automatic failover to Claude 3.5 Sonnet; CTO alerted; error_rate card turns amber |
| Budget exhausted mid-month | Pipeline auto-paused; all institutions see "AI generation temporarily unavailable"; CTO + CEO email |
| Worker hangs on single job > 10 min | Celery soft time limit triggers; job marked "failed"; re-queued once automatically |
| Quality rate < 70% for 24h | Auto-alert P1 incident: "AI MCQ quality degradation detected — possible model regression" |

---

## 12. Performance & Scaling

- Celery workers: auto-scaled 4–20 workers on `ai_generation` queue based on queue depth
- Each MCQ generation job: ~3–8s (LLM API call)
- Redis cache TTL 115s — queue depth is near-live
- Cost aggregation: `AIAPICallLog` hourly buckets — efficient monthly rollup
- Token costs computed from published provider pricing tables stored in `settings.AI_PRICING`
- Daily cost rollup Celery beat task: `aggregate_ai_costs()` at 23:55 IST
