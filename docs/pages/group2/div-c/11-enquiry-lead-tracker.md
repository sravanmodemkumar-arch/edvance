# Page 11 — Enquiry & Lead Tracker

- **URL:** `/group/adm/leads/`
- **Template:** `portal_base.html`
- **Theme:** Light

---

## 1. Purpose

The Enquiry and Lead Tracker is the entry point of the admissions funnel. Every prospective student interaction — regardless of origin — is captured here as an enquiry or lead record before it matures into a formal application. Sources include walk-in visitors to any branch, inbound WhatsApp messages, Google Form submissions from the group website or digital campaigns, school fairs and education expos, alumni referrals, and enquiries generated by demo class attendees. Without this page, the group loses visibility of interest that never made it to the application stage, and counsellors have no systematic queue to work from.

The Counsellor uses this page as their primary daily workspace. Their queue shows open leads, follow-ups due today, and leads at risk of going cold. The Coordinator monitors conversion rates across the group: how many enquiries are becoming applications, which branches are struggling to convert, and which lead sources have the highest ROI. The Director uses this data to make decisions about marketing spend and campaign targeting. The Lead Source Breakdown chart (Section 5.2) and Conversion Funnel (Section 5.4) are therefore as important to senior leadership as the Enquiry Table is to the Counsellor.

The Lost Lead Analysis (Section 5.5) closes the feedback loop. Systematically recording the reason a lead was lost — parent chose another school, unaffordable fees, wrong stream, no response — allows the group to detect patterns. If 40% of MPC enquiries are lost due to fee concerns, that is actionable intelligence. The Demo Class Coordinator also has read access to demo-sourced leads to track which demo attendees progressed to enrolment, allowing return-on-investment calculation for the demo programme.

## 2. Role Access

| Role | Level | Access | Notes |
|---|---|---|---|
| Group Admissions Director (Role 23) | G3 | Read all leads + override assignment + export | Cannot create new leads; can escalate or override Counsellor decisions |
| Group Admission Coordinator (Role 24) | G3 | Full access — all leads, all branches | Can create, assign, bulk-manage, export |
| Group Admission Counsellor (Role 25) | G3 | Own assigned leads + create new | Cannot see leads assigned to other counsellors |
| Group Demo Class Coordinator (Role 29) | G3 | Read only — demo-sourced leads | Sees only leads with `source = 'demo_class'` |
| CEO / Executive | G3+ | Read only | All leads for dashboard purposes |

> **Enforcement:** Counsellor querysets are filtered at the ORM level: `Lead.objects.filter(assigned_counsellor=request.user)`. Demo Coordinator querysets are filtered: `Lead.objects.filter(source='demo_class')`. All create and bulk-action endpoints validate role at the Django view layer.

## 4. KPI Summary Bar

> Auto-refreshes every 5 minutes via HTMX: `hx-get="/api/v1/group/{group_id}/adm/leads/kpis/" hx-trigger="every 5m" hx-target="#leads-kpi-bar" hx-swap="innerHTML"`

| Card | Metric | Source | Colour Rule | Drill-down |
|---|---|---|---|---|
| New Enquiries Today | Enquiries created today | `created_at__date=today` | Green if > yesterday; grey otherwise | Filters table to today's enquiries |
| Open Enquiries | Enquiries with status in New / Contacted / Session Scheduled | Status filter | Neutral (blue) | Filters table to open statuses |
| Follow-ups Due Today | Enquiries with `follow_up_date = today` | Date filter | Amber if > 0; green if 0 | Scrolls to Follow-up Due Today section (5.3) |
| Conversion Rate (this cycle) | `(Enquiries Converted to Application / Total Enquiries) * 100` | Computed | Green ≥ 30%; amber 15–29%; red < 15% | Opens conversion funnel (5.4) |
| Avg Enquiry Age (days) | Mean of `(today - created_at)` for open enquiries | Computed | Green ≤ 5; amber 6–10; red > 10 | Filters table to oldest enquiries |
| Top Lead Source (this month) | Source with highest new enquiry count this month | Computed | Neutral (indigo) | Filters lead source chart to top source |

## 5. Sections

### 5.1 Enquiry Table

**Display:** Sortable, selectable, server-side paginated table. 20 rows per page. Checkbox column for bulk actions. Sticky header.

**Columns:**

| Column | Notes |
|---|---|
| ☐ Checkbox | Select for bulk actions |
| Enquiry ID | Auto-generated; clickable — opens enquiry-detail drawer |
| Student Name | Clickable — opens enquiry-detail drawer |
| Class | Class student is currently in or applying for |
| Stream Interest | MPC / BiPC / MEC / CEC / General / Undecided |
| Branch | Branch the enquiry is associated with |
| Source | Badge: Walk-in / WhatsApp / Google Form / School Fair / Alumni Referral / Demo / Website / Other |
| Date | `DD MMM YYYY` of enquiry creation |
| Assigned Counsellor | Name or "Unassigned" (red text if unassigned) |
| Status | Badge: New (grey) / Contacted (blue) / Session Scheduled (indigo) / Converted (green) / Lost (red) |
| Follow-up Date | `DD MMM YYYY`; red text if date is past today |
| [Take Action] | Dropdown: `[Log Contact]` `[Schedule Session]` `[Convert to Application]` `[Mark as Lost]` |

**Search:** Live search by Name, Phone, or Email — `hx-trigger="keyup changed delay:400ms"`.

**Filters:**
- Status (multi-select)
- Source (multi-select)
- Branch (multi-select)
- Stream (multi-select)
- Date range (from / to)
- Counsellor (dropdown — Coordinator/Director only)
- Follow-up overdue (toggle — shows only enquiries where follow-up date < today)

**Bulk Actions (Coordinator / Director):**
- `[Assign Counsellor]` — bulk assign
- `[Mark as Lost]` — bulk status update with required reason
- `[Export]` — CSV of selected rows

**HTMX Pattern:** Filter/search: `hx-get="/api/v1/group/{group_id}/adm/leads/?{filters}"` `hx-target="#leads-table-body"` `hx-swap="innerHTML"` `hx-trigger="change"`. Pagination: `hx-target="#leads-table-wrapper"`. Row action click opens relevant modal/drawer.

**Empty State:** Illustration of empty funnel. Heading: "No Enquiries Found." Description: "No enquiries match your current filters, or no enquiries have been logged yet." CTA: `[+ New Enquiry]`

---

### 5.2 Lead Source Breakdown

**Display:** Donut chart (Chart.js 4.x) showing distribution of enquiries by source for the current cycle. Segments: Walk-in / WhatsApp / Google Form / School Fair / Alumni Referral / Demo / Website / Other. Each segment shows count and percentage on hover. Legend below chart. Clicking a segment filters the Enquiry Table by that source.

**HTMX Pattern:** `hx-get="/api/v1/group/{group_id}/adm/leads/source-breakdown/"` `hx-trigger="load"` `hx-target="#lead-source-chart-data"` `hx-swap="innerHTML"`; segment click: `hx-get="/api/v1/group/{group_id}/adm/leads/?source={source}"` `hx-target="#leads-table-body"` `hx-swap="innerHTML"`

**Empty State:** "No enquiry source data available for the current cycle."

---

### 5.3 Follow-up Due Today

**Display:** Highlighted sub-table pinned above the main Enquiry Table. Shows only enquiries where `follow_up_date = today`. Background tinted amber. Compact row format: Student Name | Stream | Branch | Counsellor | Last Contact | Status | `[Log Contact →]` `[Reschedule Follow-up →]`. Collapses to a count badge if empty.

**HTMX Pattern:** `hx-get="/api/v1/group/{group_id}/adm/leads/?follow_up_date=today"` `hx-trigger="load, every 5m"` `hx-target="#follow-up-due-panel"` `hx-swap="innerHTML"`

**Empty State:** Green banner: "No follow-ups due today. Well done!"

---

### 5.4 Conversion Funnel

**Display:** Horizontal funnel chart (Chart.js 4.x — rendered as a horizontal bar chart with decreasing widths or a dedicated funnel plugin). Five stages: Enquiry → Contacted → Session Scheduled → Application Submitted → Enrolled. Each stage shows absolute count and drop-off rate vs the previous stage. Colour gradient from blue (top) to green (bottom).

**HTMX Pattern:** `hx-get="/api/v1/group/{group_id}/adm/leads/funnel/"` `hx-trigger="load"` `hx-target="#conversion-funnel-chart-data"` `hx-swap="innerHTML"`

**Empty State:** "No funnel data available for the current cycle."

---

### 5.5 Lost Lead Analysis

**Display:** Table of leads marked as Lost this month. Helps identify patterns across branches and sources.

| Column | Notes |
|---|---|
| Student Name | Clickable — opens enquiry detail |
| Stream | Stream of interest |
| Branch | Branch associated |
| Source | Lead source badge |
| Lost Reason | Dropdown-selected reason: Fees / Chose Competitor / Distance / No Response / Stream Unavailable / Other |
| Lost Date | `DD MMM YYYY` |
| Counsellor | Counsellor who handled the lead |

**Filters:** Branch, Stream, Source, Lost Reason, Date range

**HTMX Pattern:** `hx-get="/api/v1/group/{group_id}/adm/leads/?status=lost&period=this_month"` `hx-trigger="load"` `hx-target="#lost-lead-table"` `hx-swap="innerHTML"`; filter changes: `hx-trigger="change"`.

**Empty State:** "No lost leads recorded this month."

## 6. Drawers & Modals

### 6.1 Enquiry Detail Drawer
- **Width:** 640px (right-side slide-in)
- **Trigger:** Click on Enquiry ID or Student Name in table
- **HTMX Endpoint:** `GET /api/v1/group/{group_id}/adm/leads/{lead_id}/detail/`

**Tabs:**

| Tab | Content |
|---|---|
| Student Info | Name, DOB, current school, class, parent/guardian details, contact number, email, address |
| Interaction History | Chronological log of all contact attempts, session bookings, notes, and status changes with timestamps and actor |
| Preferences | Stream preferences, branch preference, hostel requirement, fee expectation range, competing schools considered |
| Schedule Session | Date/time picker, session type (individual/group/online), counsellor assignment, pre-session notes |
| Convert to Application | Converts lead to a formal application record: prompts for class, stream, branch, document checklist acknowledgement |

---

### 6.2 New Enquiry Form Drawer
- **Width:** 640px
- **Trigger:** `[+ New Enquiry]` button (walk-in capture)
- **HTMX Endpoint:** `GET /api/v1/group/{group_id}/adm/leads/new/`

**Fields:** Student name, DOB, phone, email, current school, class, stream interest, branch preference, source (dropdown), hostel required (toggle), assigned counsellor, initial notes, follow-up date.

**Actions:** `[Save Enquiry]` · `[Save & Schedule Session]` · `[Cancel]`

---

### 6.3 Modal: `mark-as-lost-confirm`
- **Width:** 400px
- **Trigger:** `[Mark as Lost]` row action from `[Take Action]` dropdown in Section 5.1
- **Fields:**
  - Reason (select dropdown, required): Not interested / Enrolled elsewhere / Budget constraint / Unreachable / Duplicate enquiry / Other
  - Notes (textarea, optional, max 300 chars)
  - Notify student (checkbox, default unchecked)
- **Buttons:** `[Confirm — Mark as Lost]` (danger red) + `[Cancel]`
- **On confirm:** `hx-post="/api/v1/group/{group_id}/adm/leads/{enquiry_id}/mark-lost/"` → `hx-target="#enquiry-table"` `hx-swap="innerHTML"` → toast "Enquiry marked as lost."
- **HTMX:** `hx-get="/api/v1/group/{group_id}/adm/leads/{enquiry_id}/lost-form/"` `hx-target="#modal-body"` `hx-swap="innerHTML"`
