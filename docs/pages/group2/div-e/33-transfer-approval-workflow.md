## 1. Purpose

The Transfer Approval Workflow page is the HR Director's dedicated queue for reviewing and deciding on staff transfer requests that have been escalated by the Transfer Coordinator. Unlike the Transfer Request Manager (page 32), which shows the full queue of all requests at all stages, this page shows only requests that are specifically pending the HR Director's action — it is an actionable inbox, not a reporting view. The page is designed for fast decision-making with all relevant context available without leaving the page.

The most critical information the HR Director needs before approving a transfer is the vacancy impact. Will approving this transfer leave the source branch without a qualified teacher for a subject? Is there a replacement plan? Has the receiving branch's requirement been validated? These questions are answered by the impact assessment document — auto-generated by the system when the Transfer Coordinator escalated the request — and displayed in full within the review drawer. The HR Director should not need to navigate elsewhere to get this information.

SLA compliance is a built-in feature of this page. Emergency transfers carry a 24-hour SLA from the moment they land in the HR Director's approval queue — every emergency request shows a countdown timer. Standard transfers carry a 7-day SLA. Hardship transfers are recommended within 5 days. The page surface-areas requests that are approaching or breaching their SLA prominently, ensuring nothing is inadvertently delayed.

The HR Director has three decision options: Approve (with optional conditions such as "Transfer effective after replacement identified"), Approve with Conditions (approval is granted but contingent on a specific action the Coordinator must complete), Decline with Reason (communicated back to the branch principal), or Request More Information (puts request back to Coordinator with a specific question, pausing the SLA clock). All decisions are logged with timestamp, HR Director name, and the full decision text for audit purposes.

## 3. Page Layout

### 3.1 Breadcrumb
`Group Portal > HR & Staff > Staff Transfers > Transfer Approval Workflow`

### 3.2 Page Header
- **Title:** Transfer Approval Workflow
- **Subtitle:** Pending transfer requests requiring your decision
- **Actions (top-right):**
  - `Export Approval Log` (secondary button)
  - Badge showing total pending requests: e.g., `3 Pending`

### 3.3 Alert Banner (conditional)

| Condition | Banner | Severity |
|---|---|---|
| Emergency request SLA breached (> 24 hours) | "SLA BREACH: Emergency transfer request [ID] for [Staff Name] has exceeded the 24-hour approval window." | Red — non-dismissible |
| Emergency request approaching SLA (< 4 hours remaining) | "URGENT: Emergency transfer [ID] must be decided within [N] hours." | Amber — non-dismissible |
| Standard request SLA breached (> 7 days) | "OVERDUE: [N] standard transfer request(s) have exceeded the 7-day approval window." | Amber — dismissible |
| No pending requests | "No transfers pending your approval. All escalated requests have been resolved." | Green — auto-dismiss 10s |

## 5. Main Table — Approval Queue

| Column | Type | Sortable | Filterable |
|---|---|---|---|
| Request ID | Text | No | No |
| Staff | Text + avatar | Yes | No |
| From Branch | Badge | Yes | Yes — dropdown |
| To Branch | Badge | Yes | Yes — dropdown |
| Type | Badge (Admin / Mutual / Hardship / Emergency) | No | Yes — multi-select |
| Urgency | Badge (Emergency / High / Standard) | No | Yes — dropdown |
| Vacancy Risk | Traffic light (Green / Amber / Red) | No | Yes — dropdown |
| Impact Score | Numeric (0–10, calculated by system) | Yes | Yes — range |
| Requested Days Ago | Integer with SLA countdown for Emergency | Yes | No |
| Actions | Buttons (Review / Approve / Decline) | No | No |

### 5.1 Filters
- **Transfer Type:** multi-select
- **Urgency:** Emergency / High / Standard
- **Vacancy Risk:** Green / Amber / Red
- **Requested Days Ago:** range filter (e.g., 0–3 / 4–7 / 7+ days)

### 5.2 Search
Free-text search on Staff Name and Request ID. Debounced `hx-get` on keyup (400 ms).

### 5.3 Pagination
Server-side, 20 rows per page. Designed to keep this queue short — if it exceeds 20 rows regularly, an operational escalation alert fires. Shows `Showing X–Y of Z pending approvals`.

## 6. Drawers

### 6.1 Create
Not applicable — this page is an approval inbox only. Requests are created on page 32.

### 6.2 Review Request (Full Impact Analysis)
**Trigger:** Review button or row click
**Displays:** Full transfer request details, Transfer Coordinator's notes, Vacancy Impact Assessment report (subject coverage at source branch, qualified staff count, replacement plan if documented), staff's transfer history (have they been transferred before? how recently?), receiving branch's staffing situation (is this a genuine need?), SLA timer, and all three decision buttons at the bottom of the drawer.

### 6.3 Approve with Conditions
**Trigger:** Approve with Conditions button in review drawer
**Fields:**
- Approval Notes (textarea, required)
- Conditions (multi-select: "Effective only after replacement identified" / "HR Manager to confirm handover complete" / "Mutual consent re-confirmed" / Other)
- Custom Condition Text (text, if Other selected)
- Effective Date (date picker)
- Confirm button → records approval with conditions, notifies Transfer Coordinator to complete conditions before executing

### 6.4 Decline with Reason
**Trigger:** Decline button in review drawer
**Fields:**
- Decline Category (dropdown: Vacancy Risk Too High / No Replacement Plan / Branch Need Not Substantiated / Disciplinary Case Active / Procedural Non-compliance / Other)
- Detailed Reason (textarea, required)
- Notify Branch Principal (toggle, default on)
- Confirm Decline button → logs decision, notifies Transfer Coordinator and requesting branch principal
