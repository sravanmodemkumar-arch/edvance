## 7. Technology Notes

### Backend
- **FastAPI** — all data APIs (`/api/v1/it/...`) served by FastAPI microservices
- **PostgreSQL** — primary and only data store; no Redis, no in-memory caching
- **JWT Authentication** — all API calls require Bearer token; role claims validated server-side
- **Cloudflare R2** — object storage for all uploaded documents (warranty docs, KB images, certificates, license documents, phishing simulation assets); access via signed URLs with 15-minute expiry

### Frontend
- **Django** — server-rendered HTML templates, view routing, HTMX partial responses
- **HTMX** — all dynamic updates (table reloads, drawer loads, form submissions, auto-refresh) use HTMX attributes; no full-page reloads for interactions
- **Tailwind CSS** — utility-first styling; all components built with Tailwind classes
- **Progressive Enhancement** — pages function without JavaScript; HTMX enhances experience

### Data Access Patterns
- **Server-side pagination** throughout — all tables use `page` + `page_size` query parameters; no client-side pagination
- **No Redis** — all computed values (KPIs, aggregates, chart data) are calculated from PostgreSQL queries at request time; complex queries use PostgreSQL window functions and CTEs for performance
- **HTMX polling** — auto-refresh pages (System Health Monitor: every 2m; Ticket Manager: every 60s; Integration Health: every 5m) use `hx-trigger="every Xs"` directly on the container; no WebSockets

### Infrastructure
- **Cloudflare CDN** — static assets (JS, CSS, images) served via CDN
- **Lambda Functions** — Auth, Tenant, and Home APIs run on AWS Lambda
- **EduForge Tenant Architecture** — each branch is a tenant with isolated data but shared infrastructure; group portal queries across all tenants

### Security
- **All endpoints require authentication** — 401 returned for unauthenticated requests
- **Role-based access control (RBAC)** — enforced at FastAPI endpoint level using JWT role claims; Django views also enforce role checks
- **Audit logging** — all IT configuration changes automatically logged to `audit_log` table via Django signals; log is append-only at DB level (Page 52)
- **Data masking** — G1 roles (Role 55, 56) receive masked PII in API responses where specified; masking applied server-side

### URL Patterns
- Dashboard entry points: `/group/it/{area}/` (e.g., `/group/it/director/`, `/group/it/admin/`, `/group/it/privacy/`, `/group/it/security/`)
- Portal management: `/group/it/portals/...`
- User management: `/group/it/users/...`
- Notifications: `/group/it/notifications/...`
