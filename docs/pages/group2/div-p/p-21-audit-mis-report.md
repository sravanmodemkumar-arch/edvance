## 1. Purpose

The Audit MIS (Management Information System) Report is the formal periodic report that the Audit & Quality function submits to the Board, CEO, and Group Management Committee. It is the most important output of Division P — the document that converts all audit activity, inspection data, CAPA records, compliance scores, escalation history, and improvement plan progress into a structured management report that decision-makers can act on.

In Indian education groups, management reporting is often informal — a verbal update at the monthly principal meeting, a WhatsApp message from the audit head, or an Excel file emailed to the Chairman. None of these are audit-grade. A proper MIS report is a dated, signed, version-controlled document covering a defined period, with standardised metrics, narrative commentary, branch-level detail, and forward-looking recommendations. This page generates, stores, and manages the complete library of Audit MIS Reports — monthly, quarterly, half-yearly, and annual.

The problems this page solves:

1. **No standardised audit reporting:** Without a template, every audit report looks different. Some months the Audit Head focuses on financial findings; other months on CAPA closure; occasionally on affiliation. A standardised MIS template ensures all dimensions are covered every period with the same structure.

2. **Report generation friction:** Manually compiling audit data from Excel sheets, inspection photos, CAPA logs, and affiliation records into a Word document takes 2–3 days. The platform auto-generates the report from live data — the Analyst configures the period and sections, the system populates metrics and charts, and the Analyst adds commentary. Generation time: < 30 minutes.

3. **No report archive or version history:** Reports are emailed and forgotten. There is no searchable archive, no version history, no record of who reviewed and approved each report. The platform maintains a complete library with approval workflow and distribution tracking.

4. **Lagging performance visibility:** By the time a monthly audit report reaches the CEO (often 2–3 weeks after month end), the data is stale. The platform generates real-time MIS summaries available instantly, with the formal approved report distributed on a fixed schedule.

5. **No comparison to prior period:** A standalone report doesn't tell you if performance is improving. The MIS report includes automatic prior-period comparison for all key metrics.

**Scale:** Monthly reports (12/year) + quarterly reports (4/year) + annual report (1/year) = 17 formal reports per year · 28 branches · 6 dimensions · Distribution to 5–15 recipients

## 4. Report Generation Workflow

### Step 1 — Configure Report

**Trigger:** `[+ Generate New Report]` button.

**Configuration form:**

| Field | Type | Notes |
|---|---|---|
| Report Type | Radio: Monthly / Quarterly / Half-Yearly / Annual | |
| Period | From – To date range (auto-suggested based on type) | |
| Branches | Multi-select (default: all) | Can generate subset for zone-level report |
| Sections to Include | Checkbox list (see Section 5) | All checked by default |
| Comparison Period | Auto: prior period + prior year same period | |
| Generate Charts | ✅ Yes (default) | Auto-pulls from analytics modules |
| Auto-populate Narrative | ✅ Yes — AI-assisted draft (editable) | Key insights auto-drafted from data |

### Step 2 — Auto-Generation

System pulls data from:
- P-01 (Audit Dashboard) — KPIs
- P-02 (Audit Calendar) — audit coverage metrics
- P-03 (Financial Audit) — financial audit findings
- P-04 (Academic Quality Audit) — academic audit metrics
- P-05 (Operational/Safety Audit) — safety audit data
- P-06 (Audit Finding Tracker) — all findings
- P-07 to P-09 (Inspection) — inspection data
- P-10 (Branch Compliance Scorecard) — scores
- P-11 to P-14 (Affiliation/Regulatory) — compliance status
- P-15 (CAPA Register) — CAPA metrics
- P-16 (Escalation Manager) — escalation data
- P-17 (BIP) — improvement plan status
- P-18 to P-19 (Grievance Audit) — grievance metrics
- P-20 (Compliance Trend Analytics) — trends

Generation time: < 60 seconds for monthly report, < 3 minutes for annual.

### Step 3 — Review and Edit

Report opens in an in-platform document editor. Structure:
- Pre-populated sections with metrics, charts, and tables
- Editable narrative fields (commentary, analysis, recommendations)
- Section-level approve/reject: Analyst can exclude a section if data is insufficient
- Annotation: Highlight specific metrics for CEO attention

### Step 4 — Submit for Approval

Analyst submits to Audit Head (121) for review and approval.

Audit Head can: Approve / Return with comments.

### Step 5 — Distribute

After approval, Analyst distributes via:
- Platform email (to configured recipient list)
- PDF download for manual distribution
- Distribution tracked: who received, when, read receipt (if email)

## 6. In-Platform Document Editor

The report is edited within the platform (not in Word) using a structured editor:

**Sections panel (left):** Table of contents with section-level edit/collapse/exclude controls.

**Content area (centre):**
- Auto-populated tables and charts (view-only — sourced from live data)
- Narrative text blocks (rich text editor — bold, bullet, table, highlight)
- Commentary fields below each metric table — free text with character count
- Highlight/flag for CEO attention: right-click any metric → "Flag for Executive Summary"

**Review panel (right):**
- Current status (Draft / Submitted for Review / Approved)
- Comments from Audit Head (thread per section)
- Version history (auto-saved every 2 minutes + manual save)
- Word count, section completion status

**Toolbar:**
```
[Save Draft]  [Preview PDF]  [Submit for Review]  [Download Draft PDF]
```

## 7. Approval Workflow

```
Draft (127) → Submitted for Review (→ 121 notified) → Under Review (121 reads + comments)
→ Approved (121 approves) → Distribution Pending (127 schedules)
→ Distributed (recipients notified + read receipt tracked)
→ Archived
```

**If Audit Head returns with comments:** Report goes back to Draft status with comment thread visible. Analyst addresses comments, resubmits.

**SLA for approval:**
- Monthly report: Submitted by day 3 of next month; approved by day 5; distributed by day 7.
- Quarterly: Submitted by 5th of month after quarter end; approved by 8th; distributed by 10th.

**Approval reminder:** Auto-email to 121 if report in "Under Review" status > 48 hours without action.
