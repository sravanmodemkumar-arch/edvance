# Page: Two-Factor Authentication (2FA)
**Route:** `admin.eduforge.in/2fa`
**Auth Stage:** 3 of 5 (Level 4 & 5 only)
**Access:** Platform Admin portal only — access_level 4 or 5
**Previous:** [03-otp-verification.md](03-otp-verification.md) (OTP verified successfully)
**Next:** Home page (admin.eduforge.in/home)

---

## Overview

| Property | Value |
|---|---|
| Purpose | Second factor for high-privilege accounts. Prevents session hijacking. |
| Who sees it | All Group 1 staff with access_level 4 (Infrastructure) or 5 (Super Admin) |
| Method | TOTP — Time-based One Time Password (Google Authenticator / Authy) |
| Bypass | None. Even CEO must complete 2FA every session. |
| Session duration after 2FA | 8 hours |

## Section 2 — TOTP Entry Form

| Element | Type | Spec |
|---|---|---|
| Code label | Form label | "Authenticator code" |
| Code input | 6-box OTP input | Same component as [03-otp-verification.md](03-otp-verification.md) but no countdown timer |
| Auto-submit | Yes | After 6th digit → auto-verify |
| Verify button | Primary button | "Verify Code" — manual fallback |
| Input font | Monospace | `JetBrains Mono` — clearer for code entry |

### TOTP Validation Window

| Property | Value |
|---|---|
| Time window | Current 30s period + previous 30s period (±1 window for clock skew) |
| Code validity | Codes are time-based — each code valid for ~60 seconds maximum |
| Clock drift | Server allows ±30s drift from client time |
| Reuse | Same code cannot be used twice in same 30s window (replay protection) |

### States

| State | Trigger | UI |
|---|---|---|
| Default | Page load | Empty boxes, verify button disabled |
| Filling | User types | Auto-advance digit by digit |
| Auto-submitting | 6th digit | Overlay spinner, "Verifying..." |
| Wrong code | API 401 | Shake + red boxes. "Incorrect code. X attempts remaining." Boxes clear. |
| Max attempts (5) | 5 wrong codes | Account lock. Security alert sent to security team. "Account locked — contact Security Team." |
| Success | API 200 | Green boxes. "✅ Verified". Redirect to home. |
| Expired code | Typed but 30s passed | Re-enter: "This code has expired. Please get a new code from your app." |

## API Calls

| Action | Endpoint | Method |
|---|---|---|
| Verify TOTP code | `/api/v1/auth/2fa/verify` | POST |
| Use recovery code | `/api/v1/auth/2fa/recovery` | POST |
| Get QR code (setup) | `/api/v1/auth/2fa/setup` | GET |
| Confirm setup | `/api/v1/auth/2fa/setup/confirm` | POST |
| Get backup codes | `/api/v1/auth/2fa/backup-codes` | GET |
