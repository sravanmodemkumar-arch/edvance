## Overview

Untimed, self-paced practice from EduForge's pool of 18,42,000+ questions — filtered by subject, topic, sub-topic, difficulty, and exam. Unlike mock tests (Division C-01/C-02) which simulate exam conditions, practice mode is designed for learning: instant feedback after each question, solution visible immediately, no negative marking, no timer pressure. Students can create custom practice sessions or use AI-recommended sets targeting their weak topics. Topic tests (25 questions, 30 minutes) are a hybrid — timed but focused on a single topic.

## 6. Business Rules

- Practice questions are drawn from the same 18,42,000+ question pool used for mock tests, but with different selection logic — practice mode prioritises questions the student hasn't seen before, weighted toward their weak topics; the "Include only questions I got wrong in past tests" filter creates a powerful spaced-repetition loop where students re-encounter their mistakes until they master the concept; the question pool is shared across content partners (Group 9), so a question authored by Dr. Venkat Rao may appear in both a mock test and a practice session.

- Topic tests (25 questions, 30 min) are **unlimited for ALL students** including Free tier — this is a deliberate product decision to ensure every student can practice individual topics without hitting the 5 tests/month limit; topic tests do not count toward the mock test limit; the rationale: topic tests are the core learning tool, and gating them behind Premium would undermine EduForge's mission of accessible education; Premium monetises through full mocks, analytics, and video — not basic practice.

- Instant feedback mode (solution shown after each question) is the default for practice — this maximises learning because the student sees the explanation while the question context is fresh in memory; students can disable this ("uncheck for mini-test mode") to simulate exam conditions even in practice; in mini-test mode, solutions are shown only at the end, creating a lightweight test experience without the full mock test infrastructure.

- Question difficulty is dynamically calibrated using Item Response Theory (IRT) across all 5 crore students' attempt data — a question that 90% of students get correct is "Easy" (difficulty parameter < 0.3); a question that only 15% get correct is "Hard" (difficulty > 0.8); this calibration updates weekly as new attempt data flows in; the difficulty shown to the student ("Medium") is the human-readable label mapped from the IRT parameter; practice set generation uses the student's current ability estimate (also from IRT) to select questions at the optimal challenge level — not too easy (boring) and not too hard (frustrating).

- The "Report Error" flag on each question triggers a content review workflow in the Content Partner Portal (Group 9, Division C) — if 5+ students flag the same question within a week, it's automatically escalated to the EduForge editorial team for review; common error types: wrong answer key, incorrect solution, unclear question text, missing image; the content partner who authored the question is notified and asked to fix or confirm within 48 hours.
