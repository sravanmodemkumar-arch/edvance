## Overview

Unified catalogue of all available tests across institutions and exam domains — upcoming scheduled tests, on-demand mock tests, sectional tests, and practice tests. The catalogue dynamically filters based on the student's enrolled institutions and exam domains. Ravi sees JEE full mocks from TopRank coaching, school unit tests from Sri Chaitanya, and EAMCET mocks from the domain — all in one timeline. Suresh sees only SSC CGL, IBPS Clerk, and APPSC mocks. The calendar view helps students plan their test-taking strategy across competing schedules. Peak test windows (SSC CGL mock weekends) see 18,00,000 concurrent test-takers.

## 5. API Endpoints

| # | Method | Endpoint | Description |
|---|--------|----------|-------------|
| 1 | `GET` | `/api/v1/student/tests/upcoming?days=30` | Upcoming tests across all sources |
| 2 | `GET` | `/api/v1/student/tests/calendar?month=2026-04` | Calendar view data for a month |
| 3 | `GET` | `/api/v1/student/tests/on-demand?domain={domain}` | On-demand test library by domain |
| 4 | `GET` | `/api/v1/student/tests/{test_id}` | Test details (pattern, rules, duration) |
| 5 | `POST` | `/api/v1/student/tests/{test_id}/register` | Register for a scheduled test |
| 6 | `POST` | `/api/v1/student/tests/{test_id}/start` | Start a test (returns test session) |
| 7 | `GET` | `/api/v1/student/tests/history` | All past tests with scores |
| 8 | `GET` | `/api/v1/student/tests/recommendations` | AI-recommended tests based on weak topics |
| 9 | `POST` | `/api/v1/student/tests/{test_id}/calendar-sync` | Export test to Google Calendar / .ics |

## 6. Business Rules

- The test catalogue merges tests from three sources: (1) institution-scheduled tests (school exams, coaching mocks with fixed timing), (2) domain-scheduled tests (SSC Mega Mocks with test-window timing), and (3) on-demand tests (available anytime from the test library); scheduled tests have a start/end time or a flexible window; on-demand tests have no time constraint — the student starts when ready and the timer begins; the catalogue prioritises scheduled tests (today and tomorrow) at the top, followed by AI-recommended on-demand tests, and finally the full library.

- Free-tier students can take 5 mock tests per month across all domains combined — this limit applies to full mocks and sectional tests but NOT to topic tests (25-question short tests) which are unlimited; the counter resets on the 1st of each month; when the limit is reached, the "Start" button is replaced with "Upgrade for unlimited tests — ₹299/month" and the student can still access previous test results and analysis; Premium students have no limits.

- Test scheduling conflict detection warns students when two tests overlap — if Ravi has a school Chemistry practical at 9 AM and a JEE mock starting at 10 AM, the calendar shows a conflict icon and the timeline suggests: "Your Chemistry practical may run until 12 PM, overlapping with JEE Mock #25 (10 AM – 1 PM). Consider taking the mock in the evening window (4 PM – 7 PM) instead."; this conflict detection spans all institutions and domains.

- System check before test start verifies: browser compatibility (Chrome, Firefox, Safari, Edge — latest 2 versions), internet connectivity (minimum 2 Mbps for reliable test delivery), and screen size (warns on screens < 768px width but allows proceeding); the system check is advisory, not blocking — students in Tier-3 towns with intermittent 3G connectivity must still be able to take tests; test responses are saved to local storage every 30 seconds as a failsafe against disconnection, and auto-synced when connectivity returns.

- Tab switching during tests is tracked but not blocked — the system logs every tab-switch event with timestamp and duration; this data is shown in the test analysis ("You switched tabs 3 times for a total of 45 seconds") as a self-awareness tool; it is NOT shared with institutions or used for proctoring — EduForge mocks are practice tests, not high-stakes exams; however, institution-scheduled tests (school exams conducted online) may enable strict proctoring mode where tab-switching triggers a warning popup.
