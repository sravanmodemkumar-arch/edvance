## Why These Pages Are in This Division

Indian school student records are governed by multiple regulations:
- **CBSE Affiliation Bye-Laws:** Mandates TC format, bonafide certificate format, student register format
- **RTE Act 2009 Section 12(1)(c):** 25% EWS/DG admission quota — schools must maintain lottery records and attendance records for RTE students
- **DPDPA 2023:** Student data privacy — Aadhaar-linked records require consent, data minimisation
- **NEP 2020 — APAAR ID:** Academic Bank of Credits unique ID for every student — schools must register students
- **UDISE+:** Annual government census of school data — student counts, infrastructure, teacher data
- **CWSN (Children with Special Needs):** Schools receiving RTE grants must maintain IEP records and accommodation logs
- **POCSO Act:** Admission documents help establish student age (critical for POCSO cases)
- **State board affiliation:** Some states mandate digital student registers with e-signatures

The admission process in Indian private schools is more complex than it appears:
- Admission season (Dec–Mar) involves hundreds of enquiries, tests, and interviews
- Class XI stream allocation requires counselling records and parent consent
- RTE lottery (for aided/unaided schools under 25% quota) requires transparent documentation
- CBSE requires specific TC format with departure date, reason, last class attended, overall grade
- Bonafide certificates are needed for scholarships, bank accounts, railway passes, competitive exam applications, visa — dozens per year per school

## Key Workflows Across Pages

### Workflow 1: New Admission (Nov–Mar)
```
Enquiry (C-01) → Application (C-02) → Test/Interview (C-03) →
Merit List / Seat (C-04) → Enrollment (C-05) → Document Collection (C-06) →
Student Profile created (C-08) → ID Card (C-15) → Family Linkage (C-20)
```

### Workflow 2: RTE Admission (separate track, parallel to above)
```
RTE Application (C-07) → Income/Caste Verification → Lottery (C-07) →
Enrollment (C-05) → Document Collection (C-06) →
APAAR + Aadhaar seed (C-09) → Report to UDISE (C-17)
```

### Workflow 3: Year-End Promotion (April–May)
```
Result Computation complete (B-18) → Class Promotion Manager (C-10) →
Promote / Retain / Supplementary decisions →
Stream selection for IX→XI (C-10) → New class allocation →
Update UDISE data (C-17)
```

### Workflow 4: Student Exit
```
Parent requests TC → TC Generator (C-13) →
Principal signs TC (A-23 approval) → TC issued →
Student marked in Withdrawal Register (C-12) →
UDISE dropout count updated (C-17)
```

### Workflow 5: Certificate Request
```
Student/Parent request → Bonafide (C-14) or ID Card (C-15) →
Class Teacher/Admin generates → Certificate Log entry (C-16) →
Parent collects
```

## Why These Pages Were Added (Deep Gap Analysis)

### Gaps That Would Have Been Missed Without Deep Analysis

1. **C-07 RTE Admission Manager** — RTE Section 12(1)(c) mandates 25% EWS/DG admission in all private unaided schools. This is a legally separate admission track with lottery requirements, income verification, and annual government audit. Without this page, schools cannot document RTE compliance.

2. **C-09 APAAR & Aadhaar Linkage** — NEP 2020 mandates APAAR ID (Academic Bank of Credits ID) for every student, seeded with Aadhaar. DPDPA 2023 requires explicit consent logging for Aadhaar linkage. Without this page, schools violate both NEP implementation guidelines and data protection requirements.

3. **C-10 Class Promotion Manager** — Year-end promotion is a massive annual operation. Without a dedicated page, students remain stuck in their old class. This feeds into new timetables, fee structures, roll number generation, and hall ticket generation. Critical P0.

4. **C-12 Withdrawal & Dropout Register** — UDISE requires accurate dropout counts. RTE requires tracking of out-of-school children. Schools with high dropout rates (especially in Classes I-V) must justify or face RTE compliance flags. This register feeds OOSC (Out-of-School Children) data to government.

5. **C-16 Certificate Issuance Log** — CBSE inspection specifically checks: "How many TCs issued? What is the numbering sequence? Are all entries logged?" Schools caught issuing undocumented TCs face de-affiliation proceedings. This log provides the audit trail.

6. **C-17 UDISE Data Manager** — UDISE+ (Unified District Information System for Education) is the government's annual school census. Data submitted determines school grants, teacher allotment (government schools), and infrastructure funding. Schools that submit incorrect data lose grants. A dedicated page with validation is essential.

7. **C-19 CWSN Register** — RTE Section 31 mandates inclusion of children with disabilities in regular schools with reasonable accommodation. CBSE provides extra time and scribes to CWSN students only if the school submits the CWSN register to CBSE. Without this page, CWSN students lose their entitlements at board exams.

8. **C-21 Admission Analytics** — School management decisions on sections (how many Class I sections to open next year), teacher recruitment, and fee structure depend on understanding: enquiry conversion rate, class-wise vacancy fill rate, RTE vs general seat ratio, and year-on-year admission trend. Without this page, Principals make these decisions from memory.
